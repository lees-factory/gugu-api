# Part 1 — Problem Alignment

## Problem Statement

| 해외 쇼핑(직구·구매대행) 사용자는 현재 가격이 싼지 비싼지 판단할 근거가 없다. 가격 비교 사이트는 '지금 이 순간'만 보여줄 뿐, 시간의 흐름 속 가격 맥락을 제공하지 않는다. |
| --- |


해외 쇼핑을 하는 사람은 언제나 같은 질문을 한다: "지금 사도 되는가?" 하지만 이 질문에 답할 수 있는 도구가 없다. 가격 비교 사이트는 여러 사이트의 현재가를 나열하지만, 이 상품이 지난 3개월간 얼마였는지, 지금이 사이클의 최저점인지 최고점인지 알 방법이 없다. 결국 유저는 감에 의존한 충동구매를 하거나, 판단 자체를 포기한다. 

## Why This Matters

### 고객에게

* 잘못된 타이밍의 구매 = 직접적인 금전 손실 


* 구매대행 셀러는 소싱 타이밍이 마진을 결정함 → 데이터가 곧 경쟁력 


* 가격 맥락 없이는 '세일'인지 '정상가'인지 구분 불가 



### 비즈니스에

* Keepa(Amazon 전용)는 존재하지만 멀티 마켓 지원 도구는 없음 → 공백 시장 


* 데이터 축적 구조로 시간이 지날수록 제품 가치와 전환율이 함께 상승 


* 개인 도구 철학 → lock-in 자연 발생 → 낮은 churn 



## Supporting Evidence

* Keepa는 Amazon 전용으로 수백만 유저를 확보 — 수요 자체는 검증됨 


* 국내 직구 커뮤니티(뽐뿌, 직구갤 등)에서 '이 가격 적정한가요?' 류 게시물이 지속적으로 발생 


* 구매대행 셀러들은 스프레드시트로 가격을 수동 기록하는 경우가 많음 → 도구화 니즈 명확 


* AliExpress·Taobao·eBay 통합 추적 도구는 현재 시장에 없음 



## High Level Approach

URL 하나를 붙여넣으면, 그 상품의 가격이 매일 기록된다. 쌓인 데이터를 라인 그래프로 시각화하여 최저가·최고가·평균가·변동률을 한눈에 보여준다. '지금 사도 되는가?'라는 질문에 데이터로 답한다. 

| 문제: 가격 맥락 없음 <br>

<br> 해결 형태: URL 기반 가격 히스토리 자동 기록 + 시각화 SaaS |
| --- |


이는 Amazon 전용인 Keepa를 멀티 마켓(Amazon, eBay, AliExpress, Taobao)으로 확장한 형태이나, 깊이 경쟁이 아닌 범위 + 단순 UX + 직구러 친화로 차별화한다. 

## Narrative

### ① 직구러 지현 씨의 하루

지현 씨는 AliExpress에서 기계식 키보드를 발견했다. 가격은 $89. 싼 건지 모르겠다. 구글에 검색해봐도 현재 최저가 비교만 나오고, 3개월 전에 얼마였는지 알 수 없다. 결국 '뭐 적당한 거겠지'라며 구매한다. 일주일 후 같은 상품이 $65로 내려간다. 

**Gugu Plus가 있다면:** URL을 등록해두고, 히스토리 그래프를 보는 순간 $89가 지난 90일 최고가였다는 것을 알게 된다. 구매를 보류하고, 알림을 설정한다. 

### ② 구매대행 셀러 준혁 씨의 소싱

준혁 씨는 eBay에서 나이키 운동화를 소싱하려 한다. 지금 $120인데, 이게 평소 가격인지 일시적 할인인지 알 방법이 없다. Gugu Plus를 켜면 추적 시작 이후 60일 데이터가 쌓여있다. 평균가 $105, 최저 $95, 지금은 최고가 구간. 소싱을 미룬다. 2주 후 $98로 떨어질 때 구매한다. 

## Goals

### Primary Goals

* **G1 (측정 가능)** | 출시 3개월 내 유료 전환 유저 100명 달성 


* **G2 (측정 가능)** | Pro 유저의 60일 Retention 60% 이상 


* **G3 (측정 가능)** | 1인당 평균 추적 아이템 수 3개 이상 (데이터 축적 증거) 


* **G4 (감성 목표)** | 유저가 '내 데이터'라는 소유감을 느낌 → Export, 히스토리 강조 


* **G5 (감성 목표)** | Notion/Linear처럼 '쓰는 것 자체가 기분 좋은' UX 



## Non-goals

| Non-goal | 이유 |
| --- | --- |
| 가격 비교 / 검색 기능 | 우리는 기록 도구. 비교 플랫폼이 되면 아키텍처·BM·포지셔닝 모두 바뀜 |
| 1시간 이하 주기 업데이트 | API 쿼터·서버 비용 폭증. 가격은 일 단위로도 충분히 의미 있음 |
| 커뮤니티 / 핫딜 기능 | 다른 사업. 브랜드 톤과도 맞지 않음 |
| 광고 수익 모델 | 브랜드 훼손. 데이터 신뢰와 충돌 |
| SKU 자동 매칭 | 복잡성 폭증. URL 기반으로 충분 |
| Amazon Buy Box / 재고 추적 | Keepa와 정면 경쟁. 우리가 이길 수 없는 영역 |

| 🛑 Problem Alignment 완료 전까지 Solution 단계로 진행하지 않는다. |

# Part 2 — Solution Alignment

## Key Features (Plan of Record)

우선순위 순으로 정렬. 이것이 제품의 경계선(perimeter)이다. 

### F1 | URL 기반 상품 등록 (P0)

* 사용자가 URL을 입력하면 상품 정보와 가격을 자동 추출 


* 지원 사이트: Amazon, eBay, AliExpress, Taobao 


* 검색 기능 없음 — 링크 입력만 허용 (포지셔닝 원칙) 



### F2 | 가격 히스토리 자동 수집 (P0)

* 24시간 주기로 가격 스냅샷 자동 저장 (Free / Pro 동일) 


* 유니크 URL 기준 하루 1회만 조회 (fan-out 구조로 비용 최적화) 


* 원본 통화 + 환율 스냅샷 함께 저장 (데이터 무결성) 



### F3 | 가격 히스토리 시각화 (P0)

* 라인 그래프: x축 시간, y축 가격 


* Free: 최근 14일만 표시 / Pro: 전체 기간 + 필터 (7d / 30d / 90d / All) 


* 주기 변경 시 그래프에 마커 표시 ("Tracking frequency changed on [date]") 



### F4 | 가격 통계 — Pro 전용 (P1)

* 최저가 / 최고가 / 평균가 / 가격 변동률 (%) 


* 기간별 비교 분석 (7d vs 30d vs 90d) 



### F5 | 알림 시스템 (P1)

* Free: 이메일 알림 (가격 변동 감지 시) 


* Pro: 이메일 + 웹 푸시 알림 



### F6 | CSV Export — Pro 전용 (P1)

* 전체 히스토리 데이터 CSV 다운로드 


* 포함 컬럼: timestamp, price, currency, exchange_rate, site 


* '내 데이터' 소유감 강화 → 신뢰 + 전환 유도 



### F7 | 플랜 관리 / 구독 (P1)

* Free → Pro 업그레이드 플로우 


* Pro → Free 다운그레이드 시 데이터 잠금(삭제 없음) + 복원 안내 



## Future Considerations (Next Phase)

* 변동성 기반 자동 주기 튜닝 — 변동 없으면 느리게, 잦으면 빠르게 


* 팀 기능 — 공유 추적 리스트 


* Public API — 셀러 자동화 연동 


* 인기 상품 랭킹 / 가격 하락률 순위 (공개 데이터 집계) 


* 공유 가능한 히스토리 뷰 링크 



## Key Flows

| 4 | 대시보드에서 아이템 카드 확인 | 현재가·추적 시작일·"Data collecting..." 상태 표시 |

## Key Logic & Rules

### L1 — 주기 변경 정책

| 주기 변경은 미래부터만 적용된다. 과거 데이터는 수집 당시 주기 그대로 보존된다. 절대 인위적 보간(interpolation) 또는 synthetic 데이터 생성 금지. |
| --- |


### L2 — 데이터 수집 구조

* 유니크 URL 기준으로 하루 1회만 조회 — 동일 링크를 구독한 유저 수와 무관 


* 수집 실패 시 지수 백오프 후 재시도 (최대 3회) 


* 429 응답 시 해당 provider 큐 일시 정지 + 다음 사이클 지연 



### L3 — 히스토리 접근 권한

* Free: 최근 14일만 열람. 14일 이전 데이터는 DB에 존재하지만 잠금 


* Pro → Free 다운그레이드: 데이터 삭제 없음. 접근 제한만 적용 


* 재구독 시: 즉시 전체 기간 복원 



### L4 — 데이터 저장 원칙

* 가격 스냅샷은 반드시 원본 통화 + 당시 환율 스냅샷과 함께 저장 


* 원화 환산은 저장된 환율로 계산 — 현재 환율로 재계산 금지 (히스토리 무결성) 



### L5 — 플랜별 추적 한도

| 플랜 | 추적 한도 | 히스토리 | 통계 | 알림 | Export | 월 가격 |
| --- | --- | --- | --- | --- | --- | --- |
| **Free** | 5개 | 14일 | 기본 | 이메일 | ❌ | ₩0 |
| **Pro** | 50개 | 무제한 | 고급 | 이메일+웹푸시 | ✅ | ₩6,900 |

## FAQ

### Q: 왜 검색 기능이 없나요?

우리는 '가격 비교 플랫폼'이 아니라 '개인 가격 기록 도구'입니다. 검색을 추가하는 순간, 아키텍처·BM·브랜드 모두 바뀝니다. URL을 알고 있는 상품만 추적합니다. 

### Q: 왜 1시간 주기가 없나요?

가격은 일 단위로도 충분히 의미 있는 히스토리가 쌓입니다. 1시간 주기는 API 쿼터와 서버 비용을 수십 배 증가시키며, 1인 운영 구조와 맞지 않습니다. 장기적으로 Pro 고주기 옵션을 검토할 수 있습니다. 

### Q: Keepa랑 뭐가 다른가요?

Keepa는 Amazon 전용 초고도화 도구입니다. 우리는 Amazon, eBay, AliExpress, Taobao를 모두 지원하며, 전문 셀러가 아닌 직구러와 구매대행 셀러도 쉽게 사용할 수 있는 미니멀 UX를 제공합니다.
